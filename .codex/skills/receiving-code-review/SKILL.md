# コードレビュー受領

## 概要

コードレビューは感情的反応ではなく技術評価で扱う。

**中核原則:** 実装前に検証。推測前に質問。対人配慮より技術的正しさ。

## 反応パターン

1. READ - 反応せず全文読む
2. UNDERSTAND - 要件を言い直す（不明なら質問）
3. VERIFY - コードベース事実と照合
4. EVALUATE - このコードベースで妥当か評価
5. RESPOND - 技術的受理 or 根拠ある反論
6. IMPLEMENT - 1項目ずつ実装し都度テスト

## 禁止反応

- 儀礼的同意だけの発言
- 検証前の実装宣言
- 不明項目を放置して部分実装

## 不明瞭指摘の扱い

- 1つでも不明なら停止して明確化
- 項目は相互依存し得るため、部分理解で進めない

## 外部レビュアー指摘

実装前に確認:
- 技術的に正しいか
- 既存機能を壊さないか
- 現行実装理由と矛盾しないか
- 対応プラットフォーム全体で成立するか

誤りなら技術根拠で反論する。

## 実装順

1. 不明点を先に解消
2. Blocking → Simple → Complex の順で修正
3. 各修正を個別テスト
4. 回帰確認

## 結論

外部フィードバックは「命令」ではなく「評価対象の提案」。
検証し、問い、実装する。
