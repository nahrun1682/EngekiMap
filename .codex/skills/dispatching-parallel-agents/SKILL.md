# 並列エージェントのディスパッチ

## 概要

無関係な失敗が複数ある場合、順次調査は非効率。独立調査は並行実行する。

**中核原則:** 独立した問題領域ごとに1エージェントを割り当て、同時に進める。

## 使う条件

- 3件以上のテストファイルが別原因で失敗
- 複数サブシステムが独立に壊れている
- 問題ごとに文脈分離できる
- 共有状態がない

使わない条件:
- 失敗が関連している
- 全体状態理解が必要
- エージェント同士が干渉する

## パターン

1. 独立ドメインを特定
2. 焦点化したタスクを作成
3. 並列起動
4. 結果をレビューし統合

## 良いプロンプト

- スコープが1領域
- 必要文脈が自己完結
- 出力形式が明確

## よくあるミス

- 広すぎる依頼（全部直して）
- 文脈不足（場所不明）
- 制約不足（無関係変更）
- 出力曖昧（差分理由不明）

## 検証

エージェント完了後:
1. 各要約確認
2. 競合確認
3. 全体テスト実行
4. スポットチェック
